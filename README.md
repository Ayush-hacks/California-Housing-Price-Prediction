# 🏠 California Housing Price Prediction using Machine Learning

An end-to-end Machine Learning project that predicts California housing prices using **Random Forest Regression** and a complete **Scikit-Learn preprocessing pipeline**. This project demonstrates the entire machine learning workflow—from data preprocessing and model training to model persistence and prediction on unseen data.

---

# 📌 Project Overview

This project follows a complete machine learning pipeline to predict the **Median House Value** for houses in California.

The workflow includes:

- Loading the California Housing dataset
- Creating a Stratified Train-Test Split
- Data preprocessing using Scikit-Learn Pipelines
- Handling missing values
- Feature scaling
- One-Hot Encoding categorical features
- Training a Random Forest Regressor
- Saving the trained model and preprocessing pipeline
- Loading the saved model for inference
- Predicting house prices on new data
- Exporting predictions to a CSV file

The project automatically detects whether a trained model already exists.

- If no saved model is found, it trains a new model.
- If the model already exists, it loads the saved model and performs predictions.

# 📊 Machine Learning Workflow

## 1. Data Loading

The California Housing dataset is loaded using Pandas.

```python
housing = pd.read_csv("housing.csv")
```

---

## 2. Stratified Train-Test Split

Instead of randomly splitting the dataset, the project uses **StratifiedShuffleSplit** based on the median income category.

This ensures that both training and testing datasets maintain the same distribution of income levels.

---

## 3. Data Preprocessing

The preprocessing pipeline includes:

### Numerical Features

- Median Imputation
- Standard Scaling

### Categorical Features

- One-Hot Encoding

The entire preprocessing workflow is implemented using **Scikit-Learn Pipelines** and **ColumnTransformer**, ensuring consistency during both training and prediction.

---

## 4. Model Training

The model used in this project is:

**Random Forest Regressor**

Random Forest is an ensemble learning algorithm that combines multiple decision trees to improve prediction accuracy and reduce overfitting.

---

## 5. Model Persistence

Instead of training the model every time the program runs, the trained model and preprocessing pipeline are saved using **Joblib**.

Saved files:

```
model.pkl
pipeline.pkl
```

These files are automatically loaded during inference.

---

## 6. Prediction

During inference:

- The saved model is loaded
- Input data is preprocessed using the saved pipeline
- House prices are predicted
- Predictions are stored inside

```
output.csv
```

# ⚙ How the Project Works

### First Run

If no saved model is available:

- Dataset is loaded
- Data is preprocessed
- Random Forest model is trained
- Model is saved as `model.pkl`
- Pipeline is saved as `pipeline.pkl`

---

### Future Runs

If the model already exists:

- Saved model is loaded
- Saved preprocessing pipeline is loaded
- New data is transformed
- Predictions are generated
- Predictions are written to `output.csv`
